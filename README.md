## Descripción del proyecto

Este proyecto tiene como objetivo comparar dos formas de programación en Python: la Programación Tradicional y la Programación Orientada a Objetos (POO).

Para ello, se desarrolló un sistema básico de gestión de mascotas que permite registrar y mostrar información como:

* Nombre
* Especie
* Edad

## Programa Tradicional

El programa tradicional fue desarrollado utilizando:

* Variables
* Funciones
* Entrada de datos por teclado

En este enfoque no se utilizaron clases ni objetos.

Archivo principal:

```plaintext
programacion_tradicional/tradicional.py
```

## Programa Orientado a Objetos

El programa orientado a objetos fue desarrollado utilizando:

* Clases
* Objetos
* Atributos
* Métodos

Se implementó una clase llamada `Mascota`, la cual contiene los atributos y métodos necesarios para representar la información de una mascota.

Archivos principales:

```plaintext
programacion_poo/main.py
programacion_poo/mascota.py
```

## Reflexión

La Programación Tradicional permite desarrollar programas simples de manera rápida mediante funciones y variables. Sin embargo, cuando el sistema crece, el código puede volverse más difícil de organizar y mantener.

Por otro lado, la Programación Orientada a Objetos facilita la organización del código mediante clases y objetos. Este enfoque permite reutilizar código, representar mejor los elementos del mundo real y mejorar el mantenimiento de los programas.

En conclusión, la Programación Orientada a Objetos resulta más adecuada para proyectos grandes y organizados, mientras que la programación tradicional puede ser útil en programas pequeños y sencillos.
